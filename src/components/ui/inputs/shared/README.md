# Общие стили для инпутов

Этот модуль содержит централизованные стили для всех компонентов инпутов в системе.

## Структура

### Контейнеры

- `InputContainer` - основной контейнер для инпута
- `InputContainerWithPadding` - контейнер с отступом сверху (для DateInput, DateTimeInput)

### Лейблы

- `Label` - обычный лейбл
- `AbsoluteLabel` - абсолютно позиционированный лейбл
- `LeftLabel` - лейбл слева
- `RightLabel` - лейбл справа

### Обертки инпутов

- `InputWrapper` - основная обертка инпута с поддержкой состояний (error, success, warning, focused, readOnly)

### Стилизованные инпуты

- `StyledInput` - базовый стилизованный input элемент

### Вспомогательные тексты

- `HelperText` - обычный вспомогательный текст
- `ErrorText` - текст ошибки
- `SuccessText` - текст успеха

### Иконки и кнопки

- `IconContainer` - контейнер для иконок
- `ClearButton` - кнопка очистки
- `IconWrapper` - обертка для иконок
- `IconButton` - кнопка с иконкой

### Состояния загрузки и скелетона

- `LoadingSpinner` - спиннер загрузки
- `SkeletonEffect` - эффект скелетона

### Дополнительные элементы

- `AdditionalLabel` - дополнительный лейбл
- `ExtraText` - дополнительный текст
- `CharacterCounter` - счетчик символов
- `RequiredIndicator` - индикатор обязательности

## Использование

```tsx
import { InputContainer, Label, InputWrapper, StyledInput, ErrorText } from '../shared';

export const MyInput = () => (
  <InputContainer>
    <Label>Мой инпут</Label>
    <InputWrapper error="Ошибка">
      <StyledInput placeholder="Введите значение" />
    </InputWrapper>
    <ErrorText>Ошибка валидации</ErrorText>
  </InputContainer>
);
```

## Преимущества

1. **Централизация стилей** - все стили инпутов в одном месте
2. **Консистентность** - единообразный внешний вид всех инпутов
3. **Переиспользование** - общие компоненты для всех типов инпутов
4. **Легкость поддержки** - изменения в одном месте влияют на все инпуты
5. **Типизация** - полная поддержка TypeScript

## Миграция

Для миграции существующих инпутов:

1. Импортируйте нужные компоненты из `../shared`
2. Замените локальные styled компоненты на импортированные
3. Удалите дублирующиеся styled компоненты
4. Проверьте, что все пропсы совместимы

## Поддерживаемые состояния

- `error` - состояние ошибки
- `success` - состояние успеха
- `warning` - состояние предупреждения
- `focused` - состояние фокуса
- `readOnly` - состояние только для чтения
- `disabled` - отключенное состояние
- `loading` - состояние загрузки
- `skeleton` - состояние скелетона
